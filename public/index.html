Toy Store API Documentation

Toy Routes

GET /

This API Documentation

Example:
GET domain/

GET /api/toys

Get 10 toys

Query Parameters:
  • skip: Number of toys to skip (optional)
Example:
GET /api/toys?skip=10

GET /api/toys/count

Get total count of toys

GET /api/toys/search

Search toys by name or info

Query Parameters:
  • s: Search term (required)
  • skip: Number of toys to skip (optional)
Example:
GET /api/toys/search?s=lego&skip=10

GET /api/toys/prices

Get toys within a price range

Query Parameters:
  • min: Minimum price (required)
  • max: Maximum price (required)
  • skip: Number of toys to skip (optional)
Example:
GET /api/toys/prices?min=10&max=50&skip=20

GET /api/toys/category/:catname

Get toys by category

Path Parameters:
  • catname: Category name
Query Parameters:
  • skip: Number of toys to skip (optional)
Example:
GET /api/toys/category/action-figures?skip=10

POST /api/toys

Create a new toy (Requires authentication)

Request Body:
{
  "name": "New Toy",
  "info": "Description of the toy",
  "category": "Category",
  "price": 29.99,
  "img_url": "link to url",
  "user_id": "insert user id"
  
}

GET /api/toys/:id

Get a single toy by ID

Path Parameters:
  • id: Toy ID
Example:
GET /api/toys/5f8d0f3e1c9d440000a1d3e5

PUT /api/toys/:id

Update a toy by ID (Requires authentication)

Path Parameters:
  • id: Toy ID
Request Body:
{
"name": "Updated Toy Name",
"price": 39.99
}

DELETE /api/toys/:id

Delete a toy by ID (Requires authentication)

Path Parameters:
  • id: Toy ID

User Routes

POST /auth/register

Register a new user

Request Body:
{
"name": "John Doe",
"email": "john@example.com",
"password": "password123"
}
Validation:
  • name: string, 2-50 characters, required
  • email: valid email address, required
  • password: string, minimum 5 characters, required

POST /auth/login

Log in a user

Request Body:
{
  "email": "john@example.com",
  "password": "password123"
}
Validation:
  • email: valid email address, required
  • password: string, required

Note: On successful login, a token will be set in a cookie.

POST /auth/logout

Log out a user

This will clear the access token cookie.
